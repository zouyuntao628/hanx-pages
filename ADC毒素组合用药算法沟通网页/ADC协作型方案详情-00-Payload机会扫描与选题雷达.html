0A、Payload 情报雷达首页应该长什么样

核心逻辑：系统默认自动分析并推荐 Top 5 Payload；同时允许用户从候选池手动选择已有 payload，或者直接输入一个新 payload / 内部代号。这样既有“系统发现机会”的能力，也保留专家和客户指定方向的灵活性。

首页三条入口

入口 1：系统自动推荐 Top 5 Payload

系统定期抓取新闻、论文、专利、临床试验、公司管线、会议摘要、失败案例、交易/授权和内部实验记录，自动生成本周/本月最值得关注的 Top 5 Payload。

入口 2：从候选池手动选择

用户可以从 DXd、exatecan、SN-38、MMAE、MMAF、DM1、DM4、PBD、amanitin、免疫刺激型 payload、放射性 payload 等列表中选择。

入口 3：输入新 payload / 内部代号

如果用户关注系统还没收录的新 payload、内部 payload 或代号 payload，可以输入名称、别名、机制类别、相关 ADC、资料链接和内部备注。

0A、自动抓取和人工添加要并存

来源等级适合来源处理方式页面显示
A 自动抓取PubMed、ClinicalTrials、公司新闻 RSS、PRNewswire、Crossref、bioRxiv、部分专利数据库。定期抓取、去重、实体识别、趋势统计。已更新，可直接进入热点榜。
B 半自动导入会议 PDF、公司 PPT、行业研报、微信公众号、专家访谈摘要。人上传链接或文件，系统抽取 payload、机制、公司、适应症、结论，人再审核。待审核，审核后进入情报池。
C 人工添加内部失败结果、客户特别关注、药物不可采购、模型资源、BD 不想碰的方向。人工填写结构化表单，标注可信度和适用范围。作为公司可执行性和排除条件的重要信号。
人工添加入口必须保留：系统自动抓不到内部实验、客户 PPT、BD 私下反馈和失败结果，但这些信息往往最影响最终决策。

0A、每条情报都要变成“可行动信息”

情报雷达不是新闻列表。每条新闻、论文、专利或会议摘要后面都应该有动作按钮。

动作按钮什么时候使用下一步流向
加入 Payload 候选池发现某个 payload 近期热度上升，值得继续观察。进入 Payload 机会评分。
加入主题候选池发现的是“DXd 耐药”“payload × DDR”等主题，而不是单个 payload。进入主题趋势页或机会评估详情页。
标记为高优先级证据强、窗口期明显、公司资源也匹配。进入 P1 队列。
标记为风险信号出现毒性、临床失败、专利拥挤或竞品压力。进入安全性评估或排除条件库。
进入组合推荐payload 已经明确，目标和资源也基本清楚。进入方案一：Payload 组合推荐引擎。
暂不关注热度低、风险高、公司做不了或差异化不足。进入低优先级观察池。

0A、第一阶段 MVP 不需要太复杂

模块 1：热点分类入口

Payload 热点、机制主题热点、临床/产业热点、专利/IP 热点、安全性/失败信号。

模块 2：情报列表

标题、来源、日期、payload、主题、证据类型、系统摘要、推荐动作。

模块 3：Payload 候选池

系统自动推荐 + 人工加入，可排序、可打分、可标记 P1/P2/P3。

模块 4：进入下一步

选择一个 payload 或主题，补充目标和约束后，进入组合推荐或其他模块。

模块 5：数据来源监控

显示 PubMed、ClinicalTrials、Patents、新闻、会议摘要、内部资料是否更新。

模块 6：人工审核队列

会议 PDF、PPT、内部备注和专家建议进入审核队列后再影响评分。

如果用户输入的是未知 payload，系统应该先问什么

例如用户输入内部代号 HPX-001 payload，系统不应该直接给组合推荐，而是先用最少问题补齐上下文。

需要补充的问题可选答案 / 示例为什么要问
属于哪类机制？TOP1 inhibitor / microtubule inhibitor / DNA alkylator / immune-stimulating / unknown。决定用哪些相似 payload 做参照。
是否有结构或类似物？有结构、只有代号、类似 DXd / MMAE / PBD、完全未知。决定能否做相似性检索和公开资料补全。
是否已有单药活性数据？IC50、AUC、细胞系响应、ADC 内化、bystander effect。判断是否足够进入机会评估。
是否已有安全性顾虑？骨髓毒性、GI 毒性、神经毒性、窗口窄、未知。决定是否先进入安全性评估。
是否有目标癌种或模型？已有癌种、已有细胞系、已有 PDX / organoid、暂不确定。决定后续组合推荐能否落地实验。

一、这件事到底是在做什么

它回答的是“我们应该先选哪个 payload / 哪个主题”，而不是“选定 payload 后和谁联用”。如果 payload 都没选清楚，后面的组合推荐就会失去边界。

它会把最新热点、新闻、论文、专利、临床进展、竞品动态、毒性问题、耐药问题、未满足需求和公司资源放在一起，输出优先级排序。

层级核心问题输入输出
第 0 步：Payload 选择工具我们应该先选哪个 payload 或主题？热点、新闻、论文、专利、临床、竞品、公司资源。payload 排名、机会点、风险点、推荐切入主题、是否进入方案一。
第 1 步：Payload 组合推荐引擎选定 payload 后，它最值得和谁组合？已选 payload、业务目标、适应症方向、实验资源、排除条件。Top 3-10 组合推荐、机制解释、实验设计建议、风险提示。

二、Payload 选择雷达要看哪些信号

科学热度

看论文、机制、biomarker、耐药机制、毒性管理策略是否正在升温。

临床与产业

看 ADC 管线、临床数据、失败案例、大药企交易、授权和适应症扩展趋势。

专利与差异化

看组合使用、biomarker 人群、给药顺序、耐药逆转是否还有 IP 空间。

可组合性

看 payload 是否天然适合联用，例如 DDR、免疫、细胞周期、凋亡、外排泵路径。

安全性与转化

看 DLT、骨髓毒性、GI 毒性、神经毒性、bystander effect 和剂量窗口。

公司可执行性

看公司是否有 payload、ADC、细胞系、assay、dose matrix 能力和历史数据。

三、建议的 Payload 选择评分模型

维度建议权重怎么判断输出解释
科学热度15%论文、机制、biomarker、耐药研究是否活跃。这个 payload 为什么现在值得重新看。
产业热度15%临床、新闻、交易、管线是否活跃。是否有产业关注度和外部窗口期。
组合潜力25%是否有明确可组合机制和足够 partner 方向。是否值得进入组合推荐。
差异化空间15%是否能围绕 biomarker、耐药、适应症、给药顺序形成新故事。是否只是跟风，还是能做出自己的东西。
安全窗口15%是否存在可管理的剂量窗口和可转化可能。避免理论强但实际很难开发。
公司可执行性15%公司是否有 payload、模型、实验和数据基础。判断现在能不能马上做。
最终输出：P1 = 马上进入组合推荐；P2 = 值得继续观察或补资料；P3 = 暂不优先。

3. 数据集准备清单

数据集来源最低要求
候选 payload 池TOP1、微管、DNA 交联、新型 payload、公司已有 payload。至少包含 payload 名称、机制类别、已知毒性、代表 ADC。
