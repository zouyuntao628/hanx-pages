面向 ADC 毒素/载荷的组合用药推荐算法

本页用于和甲方确认项目边界：围绕已知 payload 或毒素，例如 exatecan，系统评估与小分子或其他毒素的组合潜力，输出少量最值得湿实验验证的组合方案。

核心输入 一个已知 payload / 毒素 / 小分子
核心输出 Top 3-10 个组合建议与实验优先级
首个示范对象 Exatecan / Topoisomerase I 抑制剂
验证方式 小规模高通量组合矩阵湿实验

方案需要先锁定的四个变量

这四个变量会直接影响算法建模、候选库筛选、评分权重和实验设计，因此建议在沟通中自然展开。

组合对象：毒素+毒素、毒素+小分子、ADC+小分子的建模方式不同。
推荐目标：增强杀伤、克服耐药、降低剂量和毒性对应不同评分权重。
候选范围：已上市药、临床药和研究级工具化合物的转化价值不同。
实验规模：可测试药物数、细胞系数和剂量矩阵决定输出 Top 3 还是更大候选池。

一、客户需求整理

业务目标

建立一个以 ADC payload/毒素为中心的组合推荐算法。给定一个已知毒素或 payload，算法从候选小分子、毒素或其他药物中筛选出最有实验验证价值的组合，并解释推荐依据。

机制互补 耐药路径阻断 毒性谱适配 剂量窗口匹配

项目难点

公开的组合湿实验数据较少，尤其是 ADC payload 相关组合数据更稀缺。因此早期不宜完全依赖机器学习黑箱模型，应先用机制、靶点、通路、毒性和药效窗口建立可解释的规则模型。

数据少 实验成本高 需要可解释 可形成独家数据

二、组合判断的核心规则

判断维度 优先推荐 不优先推荐 算法化指标
机制关系 机制不同但互补，例如造成 DNA 损伤 + 阻断 DNA 修复 两个药靶点或机制高度重叠，只是增强同一类毒性 机制互补分、机制重复惩罚
耐药路径 一个药的耐药替代路径可被另一个药阻断 两个药受同一耐药机制影响，例如共同被外排泵影响 耐药路径覆盖分
安全性 药效机制可协同，但主要剂量限制毒性尽量可分离 骨髓、神经、肝肾、胃肠道等毒性高度重叠 毒性重叠惩罚、治疗窗口分
有效区间 两个药在相近浓度区间产生作用，且临床暴露可达到 一个 1 nM 有效，另一个 1 uM 或更高才有效，窗口差异过大 IC50/EC50 比值、临床暴露裕度
实验可行性 有可购买化合物、可选细胞系、可测 biomarker 和明确读出 机制有趣但缺试剂、缺模型、缺可验证指标 实验可行性分、证据等级

三、建议的算法框架

第一阶段建议采用“知识库 + 标签体系 + 规则评分”的可解释模型。等首批湿实验数据回流后，再逐步加入机器学习模型或图模型。

1. 药物知识库 整理 payload、小分子、毒素、ADC、靶点、机制、适应症和临床阶段。
2. 通路标签 为每个药物打上细胞过程、通路、靶点、耐药机制和 biomarker 标签。
3. 组合评分 计算机制互补、耐药阻断、毒性重叠、剂量窗口和实验可行性。
4. 推荐解释 输出 Top 3-10 组合、推荐理由、风险点和建议实验设计。
5. 实验回流 用组合矩阵和 synergy score 反向校准权重，形成独家数据集。

四、核心评分模型

Combination Score = 0.30 * 机制互补分 + 0.20 * 耐药路径阻断分 + 0.15 * biomarker 选择性分 + 0.15 * 剂量窗口匹配分 + 0.10 * 临床/实验可行性分 + 0.10 * 创新性分 - 0.25 * 毒性重叠惩罚 - 0.15 * 机制重复惩罚
需要和甲方确认：评分权重是否以“发现强协同”为主，还是以“安全性和临床转化”为主。不同目标会导致推荐排序明显不同。

六、机制互补与排除规则

甲方原始需求里最核心的一句话是“找不同的通路，有用的不一样”。因此算法不能只做相似药物推荐，而要识别两个药物之间的功能关系：一个药造成压力，另一个药阻断修复、耐受、逃逸或替代通路。

直接协同杀伤 药物 A 造成核心损伤，药物 B 阻断细胞对损伤的修复或 checkpoint，例如 Topo I 抑制 + ATR/WEE1/CHK1 抑制。
耐药路径阻断 药物 B 靶向药物 A 的常见耐药通路，例如外排泵、DNA repair activation、凋亡逃逸或替代信号通路。
合成致死 两个药物分别打击肿瘤细胞依赖的互补脆弱点，尤其适合 biomarker 分层后的特定人群。
选择性增强 组合在肿瘤细胞中增强杀伤，但正常组织中不明显放大毒性，是后续临床转化更关心的类型。
组合类型 示例 算法判断 处理方式
损伤 + 修复阻断 Topo I inhibitor + PARP / ATR / DNA-PK inhibitor 机制互补，可能强协同 优先推荐，但必须加毒性重叠检查
损伤 + checkpoint 解除 Topo I inhibitor + WEE1 / CHK1 inhibitor 复制压力和细胞周期 checkpoint 互补 优先推荐，建议做 TP53/SLFN11 分层
有丝分裂压力 + 凋亡启动 MMAE + BCL2/MCL1 axis inhibitor 机制不同，可能提高死亡执行 探索推荐，需重点评估血液和心脏毒性
同机制叠加 Topo I inhibitor + 另一个 Topo I inhibitor 机制重复，信息增量低 降低优先级或排除
毒性主导重叠 微管毒素 + 强神经毒性药物 可能药效增强，但正常组织风险也增强 除非有明确优势，否则不优先

七、毒性谱和剂量窗口如何算法化

Toxicity Vector

毒性谱向量

为每个药物建立器官毒性向量，分值可从 0-5。组合时计算重叠程度，而不是只看“有没有毒性”。重点关注剂量限制毒性是否集中在同一器官系统。

toxicity_vector = { bone_marrow: 0-5, liver: 0-5, kidney: 0-5, neuro: 0-5, cardio: 0-5, GI: 0-5, ocular: 0-5, pulmonary: 0-5, skin: 0-5, immune: 0-5 }
Dose Window

有效区间匹配

客户提到“不能一个 1000 起效果，一个 1 起效果”，可以转成 potency ratio 和 exposure margin。对于 ADC payload，还要区分自由药物暴露和细胞内释放浓度。

potency_ratio = max(IC50_A, IC50_B) / min(IC50_A, IC50_B) exposure_margin = free_Cmax_or_AUC / effective_concentration
剂量区间关系 判断 建议
potency ratio < 10 两个药物有效区间接近，组合实验容易解释 优先进入剂量矩阵
10-100 可接受，但剂量设计需要覆盖足够范围 建议 8x8 矩阵，避免错过协同窗口
100-1000 匹配较差，需要临床暴露或细胞内暴露支持 谨慎推荐，先查 PK/PD 和毒性上限
> 1000 区间差异过大，实验结果可能不具备转化意义 通常不优先，除非有特殊递送或靶向释放机制
ADC 场景中不能简单使用游离 payload 的 IC50 替代 ADC 体内暴露。算法中建议单独记录 payload intrinsic potency、ADC-delivered intracellular exposure、systemic free payload exposure 和 bystander exposure。

九、算法输出应该长什么样

甲方真正需要的是可决策结果，而不是一个黑箱分数。因此每个组合建议都应包含推荐分、证据等级、机制解释、风险提示、适用 biomarker 和实验方案。

输入：Exatecan 癌种：可选，例如乳腺癌、肺癌、胃癌、卵巢癌 目标：增强杀伤 / 克服耐药 / 降低剂量 / 寻找创新组合 输出： 1. 推荐组合：Exatecan + ATR inhibitor 2. 综合评分：高 3. 机制解释：Topo I 抑制导致复制压力，ATR 抑制阻断复制压力应答 4. 风险提示：骨髓抑制和正常增殖组织毒性可能叠加 5. 适用模型：SLFN11 high、TP53 mutation、高复制压力模型 6. 实验建议：8x8 dose matrix，Bliss/Loewe/ZIP 评分，检测 gamma-H2AX 和 apoptosis 7. 决策建议：优先进入第一批湿实验

十、Exatecan 示例：首批组合方向

Exatecan 是 Topoisomerase I 抑制剂，可诱导 DNA 单链损伤、复制叉崩溃、复制压力和 DNA damage response。组合推荐应优先围绕 DNA damage response、复制压力耐受、细胞周期 checkpoint 和凋亡逃逸。

组合方向 机制逻辑 适用模型/biomarker 主要风险 优先级
Exatecan + ATR inhibitor Exatecan 造成复制压力，ATR 抑制削弱复制压力应答，促进 DNA 损伤累积。 高复制压力、TP53 突变、SLFN11 高表达、DNA repair deficient 骨髓抑制和正常增殖组织毒性可能叠加 高
Exatecan + WEE1 inhibitor Exatecan 造成 DNA 损伤，WEE1 抑制解除 G2/M checkpoint，诱导 mitotic catastrophe。 TP53 突变、高增殖、高复制压力肿瘤 细胞周期相关毒性、胃肠道和骨髓毒性 高
Exatecan + PARP inhibitor Exatecan 诱导 DNA 损伤，PARP 抑制削弱单链 DNA 修复，HRD 背景下更有潜力。 BRCA1/2 突变、HRD-high、SLFN11-high 骨髓抑制可能重叠，机制部分相关 中高
Exatecan + CHK1 inhibitor 阻断复制压力和 DNA damage checkpoint，使细胞失去修复时间窗口。 MYC-driven、TP53 deficient、高复制压力模型 正常增殖组织毒性需重点评估 中高
Exatecan + BCL2/MCL1/BCL-XL 轴抑制剂 Exatecan 产生 DNA 损伤压力，凋亡通路抑制剂增强死亡执行。 抗凋亡蛋白依赖模型，DNA 损伤后不易凋亡的肿瘤 不同靶点毒性差异大，例如血小板或心脏毒性 探索

Exatecan 机制链路

  • TOP1 inhibition
  • DNA single-strand break accumulation
  • replication fork collision / collapse
  • replication stress and DNA damage response
  • cell cycle arrest or apoptosis

潜在耐药路径

  • TOP1 expression or mutation changes
  • DNA repair pathway activation
  • replication stress tolerance increase
  • ABCB1/P-gp 等外排转运增强
  • 凋亡逃逸或 anti-apoptotic dependence

十一、湿实验验证建议

首批实验设计

  • 围绕 exatecan 选择 6-10 个候选小分子。
  • 每组做 6x6 或 8x8 剂量矩阵。
  • 优先选择 6-10 个具有 biomarker 差异的细胞系。
  • 计算 Bliss、Loewe、ZIP、HSA synergy score。
  • 同步记录 IC50 shift、细胞活性、凋亡和 DNA 损伤指标。

建议分层指标

  • TOP1 expression
  • SLFN11 high vs low
  • BRCA / HRD status
  • TP53 mutation status
  • ABCB1 / P-gp expression
  • baseline exatecan sensitivity
湿实验的价值不仅是验证单个组合，更重要的是形成 payload-centered combination 的独家数据集，用于迭代算法权重和建立后续壁垒。
实验层级 建议内容 目的
药效筛选 6-10 个组合，6-10 个细胞系，6x6 或 8x8 dose matrix 快速识别是否存在可重复协同窗口
机制验证 gamma-H2AX、cleaved caspase-3、cell cycle、replication stress marker 验证算法给出的机制解释是否成立
biomarker 分层 SLFN11、TP53、HRD、TOP1、ABCB1、baseline sensitivity 找到最适合组合的细胞背景和潜在适应症
安全性对照 正常细胞、骨髓相关细胞、肝细胞或类器官模型 初步判断协同是否只发生在肿瘤细胞中

1. 需求说明文档

3. 推荐算法原型

输入一个 payload，输出 Top 3-10 组合、解释、风险提示和实验建议。

4. Exatecan 示例报告

给出 exatecan 机制图谱、组合方向、优先级和首批实验建议。

5. 实验数据模板

支持录入剂量矩阵、细胞系、biomarker、synergy score 和实验结论。

6. 迭代方案

实验结果回填数据库，修正评分权重，扩展到 DXd、SN-38、MMAE、DM1、PBD 等 payload。

十四、需要甲方确认的问题清单

方案变量 可选范围 对算法和实验的影响
组合对象 毒素+毒素、毒素+小分子、ADC+小分子 决定暴露模型、毒性计算、候选库组织方式和实验体系。ADC+小分子需要考虑抗原表达、内吞、linker release 和 free payload 暴露；毒素+小分子更偏向机制互补和剂量窗口。
推荐目标 增强杀伤、克服耐药、降低剂量和毒性 决定评分权重。增强杀伤强调机制互补和 synergy；克服耐药强调 resistance pathway blockade；降低剂量和毒性强调 therapeutic window 和 toxicity separation。
候选药物范围 已上市药、临床阶段药物、研究级工具化合物 决定结果的转化距离。已上市药利于快速验证和临床联用设想；临床药适合创新组合；工具化合物适合机制验证和早期探索。
湿实验规模 可测试药物数量、细胞系数量、6x6 或 8x8 剂量矩阵 决定推荐输出粒度。如果实验规模有限，算法输出应收敛到 Top 3；如果平台支持高通量，可输出 Top 10-30 并做分层验证。

业务边界

  1. 首期只做 payload/毒素组合推荐，还是同时覆盖完整 ADC 药物？
  2. 组合对象优先是小分子，还是毒素之间的组合？
  3. 候选药物是否限定为已上市和临床阶段药物？是否允许工具化合物？
  4. 首个应用场景是机制探索、专利布局、临床前验证，还是 BD 项目筛选？

算法目标

  1. 排序更重视强协同，还是更重视安全性和转化可能？
  2. 是否需要按癌种输出不同组合？
  3. 是否需要结合 biomarker 分层，例如 TP53、HRD、SLFN11、ABCB1？
  4. 是否需要输出“不推荐组合”和排除原因？

数据资源

  1. 甲方已有多少 payload、毒素、小分子或组合实验数据？
  2. 是否有内部 IC50、细胞系敏感性、毒性或 PK 数据？
  3. 是否允许接入公开数据库和文献自动抽取？
  4. 数据交付形式是 Excel、数据库、API，还是内部系统？

湿实验条件

  1. 首批可以测试多少个组合、多少个细胞系、多少个剂量点？
  2. 是否能做 6x6 或 8x8 combination matrix？
  3. 是否能检测 gamma-H2AX、apoptosis、cell cycle 等机制读出？
  4. 是否有正常细胞或类器官模型用于初步安全性对照？

十五、建议对外表述

本项目拟建立一个面向 ADC payload 与小分子联用的机制驱动型组合推荐算法。算法以 payload 的作用机制、靶点通路、耐药机制、毒性谱和有效剂量窗口为核心输入，系统评估候选小分子或其他毒素与 payload 的机制互补性、耐药路径阻断能力、安全性适配度和实验可验证性，最终输出优先级最高的组合方案。

与传统小分子联用研究不同，本项目重点关注 ADC payload 场景下的药效协同和毒性谱适配，尤其考虑 payload 的细胞内释放、旁观者效应、游离毒素暴露、剂量限制毒性和耐药机制。项目可首先以 exatecan 为示范 payload，构建候选组合推荐模型，并通过少量高通量湿实验验证，形成独家组合药效数据集，进一步迭代算法权重和推荐能力。
